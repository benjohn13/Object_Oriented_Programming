HTML Templates

HTML templates are pre-built files composed of HTML pages that include integrated images, text content and support files for font styles or other things. For example, you clicked on a link that brought you to this page. This page is one of those HTML files, and the other pages you click on within this site are other HTML templates! Templates allow programmers to avoid repetition, and avoiding repetition is very important when you are working on multiple layers of code. Remember how in the beginning of the class, we talked about the five ways of thinking? One of those ways is Systems Thinking, which simply means take a big problem, and break it down into a smaller problem. HTML templates is a great way to take a big website, and break it down into smaller pieces for future work and changes. This is also another example of why you want to leave good descriptions in your code, so you can identify what you are working on easily.

Template Inheritance - template inheritance allows you to build a base “skeleton” template that contains all the common elements of your site and defines blocks that child templates can override. While this idea may seem complicated, it's very basic! Here's an example from Jinja. This template defines a simple HTML skeleton document that you might use for a simple two-column page. It’s the job of “child” templates to fill the empty blocks with content. Template inheritance is a way software developers keep things organized and avoid repetition. This also prevents from having to write thousands of lines of code, which can lead to preventing bugs.

Servers

The basic function of a server is to listen in on a port for incoming network requests, and a good demonstration of this is the interaction between a web server and browser. Although to a user the process is instantaneous, or nearly so, when he clicks a link while surfing the Web, several things are taking place behind the scenes: the request for the web page is transmitted to the corresponding web server, the server fetches and assembles the Web page and retransmits it using a protocol like HTTP. Finally, the user's browser receives the data, converts it, and displays the page to him.

What is HTTP? The Hypertext Transfer Protocol is designed to enable communications between clients and servers. HTTP works as a request-response protocol between a client and server. A web browser may be the client, and an application on a computer that hosts a web site may be the server. Two commonly used methods for a request-response between a client and server are: GET and POST. Here are some ways these two methods differ,

  • GET - Requests data from a specified resource.
    • They can be cached
    • They remain in the browser history
    • They can be bookmarked
    • They should never be used when dealing with sensitive data
    • They have length restrictions
    • They should be used only to retrieve data
  • POST - Submits data to be processed to a specified resource.
    • They are never cached
    • Do not remain in the browser history
    • Cannot be bookmarked
    • Have no restrictions on data length

If you would like to learn more about these two request methods, visit w3schools' website!

Validation

Validation basically means verifying on the server side that what we received is what we expected to see. An example of why we would want to do this is to protect our site from bad people that want to do bad things to it. So let's say we have a form on this site that allows people to leave comments on it. That's great for people who want to share their thoughts on the page. But some one could come along, and submit a bunch of junk code in the comment box and really fill up our database with garbage. Validation allows us to program a computer to handle junk, or come up with smart ways of dealing with junk.

Another thing we could do wth validation, is guide the user to a friendly experience. For example, we could create a form that requires the user to input a phone number for their profile. If the user does not provide a phone number that meets the required format, we could make sure they see an error message that guides them to put it in correctly!
