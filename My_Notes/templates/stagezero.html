Who do they think they are?! But it's not they're fault. It's those... machines. Well, if you're reading this, perhaps you can use these notes to help you find your way... a better way. The web is just a bunch of computers that communicate with each other. When you go to a web page like www.google.com, their computer sends a HTTP Request to a server. The server finds the appropriate HTML document and sends it back to the user's computer where a web browser interprets the page and displays it on the user's screen. Five Ways of Thinking
If you intend to tackle programming, you must first remember the great words of Master Yoda, "Do. Or do not. There is no try."... and these 5 ways of thinking:
  • Procedural Thinking- Procedural thinking involves creating perfectly clear and unambiguous instructions for a computer to follow.
  • Abstract Thinking- Abstract thinking means finding similarity, or as programmers would say, generality amongst seemingly different things.
  • Systems Thinking- Systems thinking happens when you break a big problem down into smaller pieces.
  • Technological Empathy- "Technological empathy" comes in many forms. For example, computer empathy is the ability to understand what a computer is, how it works, and what it's good and bad at doing.
  • Debugging (my favorite concept so far) - When a program doesn't work, it's because there is a mistake (also known as a "bug") somewhere in the computer code. Since these mistakes are an unavoidable part of programming, every good programmer has a system for fixing them and that system usually looks something like this:
    • Collect evidence (what makes this program fail?)
    • Generate theories (what may have caused this problem?)
    • Test those theories (if my theory is correct, how could I find out?)
    • Fix the problem
Terms and Definitions

Alright, by now you should have submitted "Project Zero" and gotten some feedback from HQ. They will be your point of contact during this mission. If you hold on to those first five basic ways of thinking, and dig deep inside yourself, you'll soon be at the other end of this and we'll all make it home safe. Here, take these definitions with you. The road is long and dangerous.

HTML (Hypertext Markup Language) - Think of this as the structure of your web page. If the web page was the Death Star, then HTML is the titanium alloy structures that make up all the levels, walls, floor, etc. HTML consists of tags upon tags that add up to make elements, just like titanium alloy in made of multiple metals.

CSS (Cascading Style Sheets) - Think of this as the "style" of your Death Star. What shade of grey is the floor? What shade of grey are the walls? Is Emperor Palpatine's chair Corinthian or Noobian grey? The options for style are limitless.

JavaScript - While you haven't touched much on this yet, you can think of this as the "interactive functions" of your Death Star... Like, you can push this button here, issue a voice command and a legion of men will follow your order.... You may fire at will, commander.

DOM (Document Object Model) - This term refers to your HTML structure. It will be comprised of multiple tags added together, making up multiple elements that form a "tree-like" structure in your coding. Think of it like multiple atoms coming together to make up a molecule, then molecules coming together to make up an element! In a browser, this structure shows up as a series of nested boxes. There are boxes inside of boxes inside of boxes, and so on...

Your Equipment

Text Editors - When writing code, programmers use special text editors (like Sublime Text for example). These editors make the programmer's life easier. For example, some text editors will automatically generate a closing HTML tag when you write an opening tag.

Developer Tools - HTML elements are either inline or block. Block elements form an "invisible box" around the content inside of them.
